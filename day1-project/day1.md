# 📅 Day 1 – DevOps Challenge (Ubuntu Setup & Git Basics)

## 🎯 Objective

Set up a DevOps-ready environment using Ubuntu and understand basic Linux and Git workflow.

---

## ⚙️ Environment Setup

* Installed Ubuntu on VMware
* Verified terminal functionality
* Updated system packages using:

  * `sudo apt update`
  * `sudo apt upgrade`

---







## 🛠️ Tools Installed

* Git (Version control system)
* Curl (for data transfer and API testing)

---

## 💻 Linux Commands Practiced

### File & Directory Management

* Created directories using `mkdir`
* Navigated using `cd`
* Listed files using `ls` and `ls -l`

### File Operations

* Created files using `touch`
* Edited files using `echo`
* Viewed content using `cat`
* Copied files using `cp`
* Renamed files using `mv`
* Deleted files using `rm`

---

## 🌿 Git Setup

* Installed Git
* Configured Git user details:

  * `git config --global user.name`
  * `git config --global user.email`

---

## 📁 Project Work

* Created project directory: `devops-challenge/day1-project`
* Created file: `app.txt`
* Added sample content to file

---

## 🔄 Git Workflow Practiced

* Initialized repository using `git init`
* Added files using `git add .`
* Committed changes using:

  * `git commit -m "Day 1: Initial project setup"`

---

## 🧠 Key Learnings

* Understood basics of DevOps workflow
* Learned how Linux is used for system operations
* Understood Git as a version control system
* Performed first commit in a local repository

## ✅ Outcome

* Ubuntu environment ready for DevOps practice
* Git installed and configured
